2-1 虚拟机的安装
1. VMware简介
VMware是一个虚拟PC的软件，可以在现有的操作系统上虚拟出一个新的硬件环境，相当于模拟出一台新的PC，
依次来实现在一台机器上真正同时运行两个独立的操作系统。
使用VMware8，占用空间较小。

VMware主要特点：
    -不需要分区或重新开机就能在同一台PC上使用两种以上的操作系统
    -本机系统可以与虚拟机系统网络通信
    -可以设定并且随时修改虚拟机操作系统的硬件环境
    虚拟机有快照功能，如果系统崩溃了，系统可以恢复到快照时的状态。

2、建议的VMware配置
    CPU：建议主频为1GHz以上
    内存：建议1GB以上
    硬盘：建议分区空闲空间8G以上

2-3 系统分区之分区与格式化
1. 磁盘分区
    磁盘分区是使用分区编辑器（partition editor）在磁盘上划分几个逻辑部分。（不是物理上的划分，只是概念上的分区）
    碟片一旦划分成数个分区（Partition），不同类的目录与文件可以存储进不同的分区。

    分区类型
        主分区：最多只能有4个。
        扩展分区：
            做多只能有1个。
            主分区加扩展分区最多有4个。
            扩展分区不能写入数据，只能包含逻辑分区。
        逻辑分区

2. 格式化
    格式化（高级格式化）又称逻辑格式化，它是指根据用户选定的文件系统（如FAT16、FAT32、NTFS、EXT2、EXT3、EXT4等），
    在磁盘的特定区域写入特定数据，在分区中划出一片用于存放文件分配表、目录表等用于文件管理的磁盘空间。

    NTFS支持更大的空间和更大的单个文件大小。
    格式化就是写入文件系统，它包括两部分，一是把磁盘分为等大小的数据块；二是在分区中建立二维表格，记录每个文件的ID号、修改时间和权限。
    默认数据块（block）的大小是4kB。
    格式化会将数据清空，这只是它的效果，它的目的是写入文件系统。

2-4 分区之分区设备文件名与挂载
3. 硬件设备文件名
    硬件                      设备文件名
    IDE硬盘                   /dev/hd[a-d]
    SCSI/SATA/USB硬盘         /dev/sd[a-p]
    光驱                      /dev/cdrom或/dev/hdc
    软盘                      /dev/fd[0-1]
    打印机（25针）             /dev/lp[0-2]
    打印机（USB）              /dev/usb/lp[0-15]
    鼠标                      /dev/mouse

    /表示根目录
    dev目录下存放的硬件设备文件

    分区设备文件名
        设备文件名
            /dev/hda1（IDE硬盘接口）
            /dev/sda1（SCSI硬盘接口、SATA硬盘接口）

    分区 --> 格式化 --> 分区设备名 --> 挂载点（相当于Windows的盘符）
    1、2、3、4四个分区号只能给主分区，逻辑分区是从5开始的。

4、挂载
    必须挂载
        /（根分区）
        swap分区（交换分区，内存2倍，不超过2GB）
    推荐分区
        /boot（启动分区，200MB）

总结：
分区：把大硬盘分为小的逻辑分区
格式化：写入文件系统
分区设备文件名：给每个分区定义设备文件名
挂载：给每个分区分配挂载点

3-1 Linux命令基本格式
[root@localhost ~] #
    root            当前登录用户
    localhost       主机名
    ~               当前所在目录（家目录），每个用户都有自己的家目录，root-/root, user1-/home/user1
    #               超级用户的提示符
                    普通用户的提示符是$

pwd -- 当前目录
cd -- 切换目录

命令格式
    命令 [选项] [参数]
注意： 个别命令使用不遵循此格式
      当有多个选项时，可以写在一起，简化选项与完整选项，选项可以调整命令的功能
        -a 等于 --all，并不是所有命令都支持 -a 选项


查询目录中内容：ls
ls [选项] [文件或目录]
选项：
    -a 显示所有文件，包括隐藏文件（all）
    -l 显示详细信息 （long）
    -d 查看目录属性
    -h 人性化显示文件大小
    -i 显示inode（文件的id号）

[root@localhost bin]# ls -l
dr-xr-xr-x.   2 root root  4096 10月  5 04:55 bin
d 文件类型（- 文件， d 目录， l 软链接文件）
    Linux一共有7种文件，还包括块设备文件、字符设备文件、套接字文件和管道文件，通过权限的第一个字符区分
r-x，u所有者的权限
r-x，g所属组的权限（相同身份或者相同权限的人分为一组）
r-x，o其他人的权限
r-read-4
w-write-2
x-execute-1

dr-xr-xr-x（权限）.   2（引用计数） root（所有者） root（所属组）  4096（文件大小） 10月  5 04:55（最后一个修改时间） bin（文件名）
4096-文件大小， ls -lh（h代表human，为了人们更好的理解数据的含义，只要包含文件大小的命令均可以使用-h命令）

[选项]，是为调整命令的功能的。
[root@localhost bin]# ls -lh cat
-rwxr-xr-x. 1 root root 44K 11月 22 2013 cat
[root@localhost bin]# ls（命令） -lh（选项） cat（参数）

Linux中以.开头的文件是隐藏文件，一般是系统文件，可以提示用户不要轻易修改它。

[root@localhost bin]# ls -i


ll 是 ls -l 命令的别名

目录处理命令

快捷键：Ctrl + L    清屏，clear


相对路径：参照当前所在目录，进行查找，如 cd ../usr/local/src/
绝对路径：从根目录开始指定，一级一级递归查找。在任何目录下，都能进入指定位置，如 cd /home/

删除文件或目录：rm
    rm -rf 文件或目录
        remove
        选项：
            -r  删除目录（删除目录时使用，删除文件时也可以使用）
            -f  强制（不用询问是否确定删除，直接删除）

            rm -rf / 自杀式命令，将会删除绝大数文件，Linux的root权限非常大

    cp [选项] [原文件或目录] [目标目录]
        copy
        选项：
            -r  复制目录
            -p  连带文件属性复制（创建时间等）
            -d  若原文件是链接文件，则复制链接属性
            -a  相当于-pdr，目标文件和原文件一模一样

剪切或改名命令：mv
    mv [原文件或目录] [目标目录]
        move
        如果原文件和目标文件在同一个目录，就是改名功能
        如果原文件和目标文件不在同一个目录，就是剪切

3-4 Linux链接命令
链接命令：ln

ln -s [原文件] [目标文件]
    link
    功能描述：生成链接文件
    选项：-s   创建软链接（soft）

硬链接与软链接：
硬链接特征：（一个真实文件的多个别名）
1 拥有相同的i节点和存储block块，可以看做是同一个文件
2 可通过i节点识别
3 不能跨分区（可以把硬链接看做一个教室的两个门，这个教室不能同时在两栋楼中）
4 不能针对目录使用

软链接特征：（推荐使用软链接）
1 类似windows快捷方式
2 软链接拥有自己的I节点和Block块，但是数据块中只保存原文件的文件名和I节点号，并没有实际的文件数据
3 lrwxrwxrwx l 软链接
    软链接文件权限都为 rwxrwxrwx
4 修改任意文件，另一个都改变
5 删除原文件，软链接不能使用

创建软链接时，原文件一定要写绝对路径，除非和目标文件在同一个目录下

4-1 文件搜索命令locate
locate命令格式
locate 文件名
    在后台数据库中按文件名搜索，搜索速度更快
    /var/lib/mlocate，是locate命令所搜索的后台数据库，不用遍历整个文件系统
    updatedb，强制更新数据库，默认一天更新一次

[root@localhost home]# locate liyazhou
locate: can not stat () `/var/lib/mlocate/mlocate.db': 没有那个文件或目录
[root@localhost home]# updatedb
[root@localhost home]# locate liyazhou

优点，搜索速度快
缺点，只能按照文件名称搜索，功能单一，与find互补
一般情况下，按文件名搜索情况较多，所以locate较常用

4-2 命令搜索命令
命令搜索命令 whereis
whereis 命令名

whereis 命令

cd 是shell的内置命令（自带），它没有执行文件

4-3 find命令
文件搜索命令find
功能强大，参数较多，学习起来复杂，此处仅仅学习常用的操作

find [搜索范围] [搜索条件]
    find / -name install.log
    避免大范围搜索，会非常耗费系统资源
    find是在系统中搜索符合条件的文件名。如果需要匹配，使用通配符匹配，通配符是完全匹配。

[root@localhost ~]# find / -name liyazhoutemp
/home/liyazhoutemp

[root@localhost ~]# find / -name liyazhou*
/home/liyazhoutemp

Linux中的通配符
    *   匹配任意内容
    ?   匹配任意一个字符
    []  匹配中括号内任意的一个字符

不区分大小写
find /root -iname install.log

按照所有者搜索
find /root -user root

查找没有所有者的文件
find /root -nouser
没有所有的文件，一般是垃圾文件
一是，内核产生的文件可能没有所有者，比如在/sys
二是，外来的文件，可能没有所有者
除了两者之外，没有所有者的文件可以看做是垃圾文件

查找10天前修改的文件（处理日志文件常用）
find /var/log/ -mtime +10
    -10     10内修改的文件
     10     10天当前修改的文件
    +10     10天前修改的文件

    atime   文件访问时间
    ctime   改变文件属性
    mtime   修改文件内容

查找文件大小是25kB的文件
find . -size 25k
    -25k    小于25kB的文件
    25k     等于25kB的文件
    +25     大于25kB的文件
单位
KB——k（小写）
MB——M（大写）

查找inode是262422的文件
find . -inum 262422
    跟 ls -i 配合使用
    ls -i 查看inode
    find dir -inum inodeid 已知文件编号查文件


查找/etc/目录下，大于20KB并且小于50KB的文件
find /etc -size +20k -a -size -50k
    -a and 逻辑与，两个调价都满足
    -o or  逻辑或，两个条件满足一个即可


查找/etc/目录下，大于20KB并且小于50KB的文件，并显示详细信息
find /etc -size +20k -a -size -50k -exec ls -lh {} \;
    对搜索结果执行操作 -exec/-ok 命令 {} \;

4-4 grep命令
搜索字符串命令 grep

在文件当中匹配符合条件的字符串
grep [选项] 字符串 文件名
    选项：
        -i 忽略大小写
        -v 排除指定字符串

[root@localhost ~]# grep "size" anaconda-ks.cfg

find命令和grep命令的区别
find命令，在系统当中搜索符合条件的文件名，如果需要匹配，使用通配符匹配，通配符是完全匹配。
grep命令，在文件当中搜索符合条件的字符串，如果需要匹配，使用正则表达式进行匹配，正则表达式时包含匹配。
